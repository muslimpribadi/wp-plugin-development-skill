# Tutorial: Building a Custom Block Editor

## Project Objective
Create a standalone block editor instance within a custom WordPress admin page using the `@wordpress/block-editor` package. This allows the Gutenberg editing experience to be used outside of the standard post/page editor.

**Target Features:**
-   Ability to add and edit Core blocks.
-   Familiar visual styles (main/sidebar layout).
-   Basic block persistence between page reloads (via LocalStorage).

---

## 1. Plugin Setup & Organization

### Directory Structure

| Path | Purpose |
| :--- | :--- |
| `plugin.php` | Main entry point; loads `init.php`. |
| `init.php` | Handles menu registration and asset enqueuing. |
| `src/` | Source code (JSX/Sass) for the editor UI. |
| `build/` | Compiled output (JS/CSS) generated by Webpack. |
| `webpack.config.js` | Custom config extending `@wordpress/scripts` for Sass support. |

### Initialization (`init.php`)
The plugin creates a custom admin page and enqueues the necessary assets to render the React application.

## 2. Server-Side Implementation (PHP)

### Step A: Register Admin Page
Use `add_menu_page()` to create the container for the editor.

```php
// File: init.php
add_menu_page(
    'Standalone Block Editor',       // Title
    'Block Editor',                  // Menu Label
    'edit_posts',                    // Capability
    'getdavesbe',                    // Slug
    'getdave_sbe_render_block_editor',// Render Callback
    'dashicons-welcome-widgets-menus'// Icon
);
```

### Step B: Output HTML Placeholder
The React app mounts into this specific `div`.

```php
// File: init.php
function getdave_sbe_render_block_editor() {
    echo '<div id="getdave-sbe-block-editor">Loading Editor...</div>';
}
```

### Step C: Enqueue Assets
Hook into `admin_enqueue_scripts` to load scripts only on this specific page (`toplevel_page_getdavesbe`).

```php
// File: init.php
function getdave_sbe_block_editor_init( $hook ) {
    if ( 'toplevel_page_getdavesbe' !== $hook ) return;

    // 1. Enqueue Main JS Bundle (from build/)
    // Dependencies are handled by @wordpress/dependency-extraction-webpack-plugin
    wp_enqueue_script( 
        'getdave-sbe-script', 
        plugins_url( 'build/index.js', __FILE__ ), 
        array(), 
        filemtime( __DIR__ . '/build/index.js' ) 
    );

    // 2. Enqueue CSS (depends on wp-edit-blocks for default styles)
    wp_enqueue_style(
        'getdave-sbe-styles',
        plugins_url( 'build/index.css', __FILE__ ),
        array( 'wp-edit-blocks' )
    );

    // 3. Inline Editor Settings (JSON -> window.getdaveSbeSettings)
    $settings = getdave_sbe_get_block_editor_settings();
    wp_add_inline_script( 
        'getdave-sbe-script', 
        'window.getdaveSbeSettings = ' . wp_json_encode( $settings ) . ';' 
    );
}
add_action( 'admin_enqueue_scripts', 'getdave_sbe_block_editor_init' );
```

## 3. Client-Side Implementation (JavaScript)

### Step A: Entry Point (`src/index.js`)
Mount the React app once the DOM is ready.

```javascript
// File: src/index.js
import { createRoot } from 'react-dom/client';
import domReady from '@wordpress/dom-ready';
import { registerCoreBlocks } from '@wordpress/block-library';
import Editor from './editor';

domReady( () => {
    const root = createRoot( document.getElementById( 'getdave-sbe-block-editor' ) );
    
    // 1. Load settings from global window object
    const settings = window.getdaveSbeSettings || {};
    
    // 2. Register Core Blocks (required for them to appear)
    registerCoreBlocks();

    // 3. Render Editor
    root.render( <Editor settings={ settings } /> );
} );
```

### Step B: The `<Editor>` Wrapper (`src/editor.js`)
This component scaffolds the layout (Header, Sidebar, Notices).

```javascript
// File: src/editor.js
import { navigateRegions } from '@wordpress/components';
import Notices from 'components/notices';
import Header from 'components/header';
import Sidebar from 'components/sidebar';
import BlockEditor from 'components/block-editor';

function Editor( { settings } ) {
    return (
        <div className="getdavesbe-block-editor-layout">
            <Notices />
            <Header />
            <Sidebar />
            <BlockEditor settings={ settings } />
        </div>
    );
}

export default navigateRegions( Editor ); // Adds keyboard navigation
```

## 4. The Core `<BlockEditor>` Component

Located in `src/components/block-editor/index.js`. This is the heart of the editor.

### Hierarchy & Props

| Component | Role | Key Props/Context |
| :--- | :--- | :--- |
| **`<BlockEditorProvider>`** | Context Provider | `value` (blocks), `onInput`, `onChange`, `settings`. |
| **`<Sidebar.InspectorFill>`** | Slot/Fill | Allows `<BlockInspector>` to render in the main Sidebar. |
| **`<BlockCanvas>`** | Visual Area | The editable area where blocks are rendered. |

### Block List Internals
The visual rendering of blocks relies on a specific hierarchy:
1.  **`<BlockList>`**: Loops over block `clientIds`.
2.  **`<BlockListBlock>`**: Renders individual block wrappers.
3.  **`<BlockEdit>`**: Calls the block's `edit()` function (defined in its metadata).

### Persistence Implementation
To save blocks between reloads, the component uses React state and `localStorage`.

```javascript
// File: src/components/block-editor/index.js
import { useState, useEffect } from '@wordpress/element';
import { serialize, parse } from '@wordpress/blocks';

function BlockEditor( { settings } ) {
    // 1. State Management
    const [ blocks, setBlocks ] = useState( [] );

    // 2. Load Data on Mount
    useEffect( () => {
        const stored = localStorage.getItem( 'getdavesbeBlocks' );
        if ( stored ) {
            setBlocks( parse( stored ) );
        }
    }, [] );

    // 3. Save Data on Change
    const persistBlocks = ( newBlocks ) => {
        setBlocks( newBlocks ); // Update React State
        localStorage.setItem( 
            'getdavesbeBlocks', 
            serialize( newBlocks ) // Convert to Block Grammar string
        );
    };

    return (
        <BlockEditorProvider
            value={ blocks }
            onInput={ setBlocks }       // Updates state immediately
            onChange={ persistBlocks }   // Handles persistence
            settings={ settings }
        >
            <Sidebar.InspectorFill>
                <BlockInspector />
            </Sidebar.InspectorFill>
            <BlockCanvas height="400px" />
        </BlockEditorProvider>
    );
}
```

## Key Takeaways

1.  **`registerCoreBlocks()`**: Essential to make standard blocks (Paragraph, Heading, etc.) available in your custom editor.
2.  **`<BlockEditorProvider>`**: The central component that manages the "state" of the editor and provides context to children.
3.  **Slot/Fill (`Sidebar.InspectorFill`)**: Used to render the Inspector (Settings Sidebar) inside the main layout while keeping it logically connected to the Editor Provider.
4.  **Serialization**: Use `serialize()` to save blocks as a string (Block Grammar) and `parse()` to restore them from storage.
